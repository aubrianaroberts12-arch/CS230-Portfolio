# CS 230 Module Eight Journal

## The Gaming Room Client and Software Requirements

The client for this project was The Gaming Room. They wanted to expand their game, Draw It or Lose It, from only running on Android devices to a web-based application that could run on different operating systems. The application needed to support multiple teams and players at the same time while preventing duplicate games from being created. It also needed to be secure, reliable, and able to grow as more users joined the game.

## What I Did Well

I think I did a good job explaining the different operating systems and recommending the best solution for the client. I also spent time making sure my recommendations matched the client's needs instead of just listing facts about each platform. As the projects went on, I felt like my software design document became more organized and easier to understand.

## How the Design Document Helped

Working through the design document before writing code helped me understand how all the parts of the application fit together. It made me think about things like system architecture, memory management, storage, security, and how users would interact with the software before actually building it. Planning everything first would make coding much easier because many of the important decisions have already been made.

## What I Would Revise

If I could revise one part of my work, I would spend more time on the system architecture section. I think I understand it much better now than I did at the beginning of the course. I would include more details about how the client, server, database, and network communicate with each other and maybe add a diagram to make it easier to understand.

## Interpreting the User's Needs

I tried to focus on what The Gaming Room actually wanted instead of designing the software the way I thought it should work. Their biggest goals were supporting multiple platforms, preventing duplicate games, allowing many users to play together, and keeping user information secure. Those needs influenced my recommendations for using Linux, Java, MySQL, and a distributed web-based design. Considering the user's needs is important because software should solve the client's problem instead of creating new ones.

## My Approach to Software Design

My approach was to first understand the client's requirements and then compare different technologies that could meet those requirements. I looked at the advantages and disadvantages of different operating systems and considered things like scalability, security, storage, and performance before making recommendations. In the future, I would continue using this process by gathering requirements first, creating a design before coding, and thinking about future growth so the software can continue to meet the user's needs as it expands.
